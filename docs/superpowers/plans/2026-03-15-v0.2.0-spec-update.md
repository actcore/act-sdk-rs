# ACT SDK v0.2.0 Spec Update Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Update act-sdk-rs to match the ACT v0.2.0 spec: remove `config` parameter, remove `get-info`/`get-config-schema`, add `get-metadata-schema`, pass metadata via `tool-call.metadata`.

**Architecture:** The WIT submodule (`wit/deps/act-core`) already has v0.2.0 WIT. We update the world.wit files to reference `@0.2.0`, update the proc macro to generate the new trait impl, update act-types to replace `Config` with metadata patterns, and update examples.

**Tech Stack:** Rust, wit-bindgen, proc_macro2/syn/quote, schemars, serde, ciborium

## Task 2: Update proc macro — remove `get-info` and `get-config-schema`

**Files:**
- Modify: `act-sdk-macros/src/component.rs`

The v0.2.0 `tool-provider` trait no longer has `get_info()` or `get_config_schema()`. It has `get_metadata_schema(metadata)` instead. `call-tool` no longer takes `config`.

- [ ] **Step 1: Update the generated trait impl in `component.rs`**

Replace the `impl exports::act::core::tool_provider::Guest` block. Remove `get_info()` and `get_config_schema()`. Add `get_metadata_schema()`. Change `list_tools` signature. Change `call_tool` signature.

In `component.rs`, replace the generated output (inside `generate()`) from line ~103:

```rust
    let output = quote! {
        // WIT bindings generation
        wit_bindgen::generate!({
            path: "wit",
            world: "component-world",
            generate_all,
        });

        // User-defined items from the module body
        #(#user_items)*

        // Generated hidden arg structs
        #(#arg_structs)*

        /// Convert a RawStreamEvent to a WIT StreamEvent.
        fn __raw_to_wit(raw: ::act_sdk::context::RawStreamEvent) -> act::core::types::StreamEvent {
            match raw {
                ::act_sdk::context::RawStreamEvent::Content { data, mime_type, metadata } => {
                    act::core::types::StreamEvent::Content(act::core::types::ContentPart {
                        data,
                        mime_type,
                        metadata,
                    })
                }
                ::act_sdk::context::RawStreamEvent::Error { kind, message, default_language: _ } => {
                    act::core::types::StreamEvent::Error(act::core::types::ToolError {
                        kind,
                        message: act::core::types::LocalizedString::Plain(message),
                        metadata: vec![],
                    })
                }
            }
        }

        struct __ActComponent;

        export!(__ActComponent);

        impl exports::act::core::tool_provider::Guest for __ActComponent {
            async fn get_metadata_schema(
                _metadata: Vec<(String, Vec<u8>)>,
            ) -> Option<String> {
                #config_schema_expr
            }

            async fn list_tools(
                _metadata: Vec<(String, Vec<u8>)>,
            ) -> Result<act::core::types::ListToolsResponse, act::core::types::ToolError> {
                Ok(act::core::types::ListToolsResponse {
                    metadata: vec![],
                    tools: vec![
                        #(#tool_defs),*
                    ],
                })
            }

            async fn call_tool(
                call: act::core::types::ToolCall,
            ) -> wit_bindgen::rt::async_support::StreamReader<act::core::types::StreamEvent> {
                let (mut writer, reader) = wit_stream::new::<act::core::types::StreamEvent>();

                wit_bindgen::spawn(async move {
                    let __default_lang = #default_lang;
                    match call.name.as_str() {
                        #(#call_arms)*
                        __other => {
                            let _ = writer.write_all(vec![
                                act::core::types::StreamEvent::Error(act::core::types::ToolError {
                                    kind: ::act_sdk::constants::ERR_NOT_FOUND.to_string(),
                                    message: act::core::types::LocalizedString::Plain(format!("Tool '{}' not found", __other)),
                                    metadata: vec![],
                                })
                            ]).await;
                        }
                    }
                });

                reader
            }
        }
    };
```

Key changes:
- Removed `fn get_info()`
- Removed `fn get_config_schema()`
- Added `async fn get_metadata_schema(_metadata: Vec<(String, Vec<u8>)>) -> Option<String>`
- `list_tools` takes `_metadata: Vec<(String, Vec<u8>)>` instead of `_config: Option<Vec<u8>>`
- `call_tool` takes only `call: act::core::types::ToolCall` (no config)

- [ ] **Step 2: Update config_schema_expr generation**

The `config_schema_expr` currently generates a schema from the config type. Rename the variable to `metadata_schema_expr` for clarity. The logic stays the same — it generates a JSON Schema from the config type if one exists.

In `generate()`, rename:
```rust
    let metadata_schema_expr =
        if let Some(config_type) = tools.iter().find_map(|t| t.config_type.as_ref()) {
            quote! {
                Some(::act_sdk::__private::serde_json::to_string(
                    &::act_sdk::__private::schemars::schema_for!(#config_type)
                ).unwrap_or_else(|_| r#"{"type":"object"}"#.to_string()))
            }
        } else {
            quote! { None }
        };
```

And use `#metadata_schema_expr` in `get_metadata_schema`.

- [ ] **Step 3: Update `gen_call_arm` — remove config handling**

In `gen_call_arm`, the `ctx_setup` block currently deserializes `_config`. Since config is now in metadata, and the SDK config pattern needs rethinking (see Task 3), for now change the context setup to extract config from `call.metadata`:

Replace the `ctx_setup` block for config_type case:
```rust
    let ctx_setup = if tool.has_context {
        if let Some(config_type) = &tool.config_type {
            quote! {
                let __config: #config_type = {
                    // Extract config from well-known metadata keys
                    let __metadata_map: ::act_sdk::__private::serde_json::Value = {
                        let mut map = ::act_sdk::__private::serde_json::Map::new();
                        for (k, v) in &call.metadata {
                            if let Ok(val) = ::act_sdk::cbor::from_cbor::<::act_sdk::__private::serde_json::Value>(v) {
                                map.insert(k.clone(), val);
                            }
                        }
                        ::act_sdk::__private::serde_json::Value::Object(map)
                    };
                    match ::act_sdk::__private::serde_json::from_value(__metadata_map) {
                        Ok(v) => v,
                        Err(e) => {
                            let _ = writer.write_all(vec![
                                act::core::types::StreamEvent::Error(act::core::types::ToolError {
                                    kind: ::act_sdk::constants::ERR_INVALID_ARGS.to_string(),
                                    message: act::core::types::LocalizedString::Plain(format!("Failed to deserialize metadata: {}", e)),
                                    metadata: vec![],
                                })
                            ]).await;
                            return;
                        }
                    }
                };
                let mut __ctx = ::act_sdk::ActContext::__new(__config, __default_lang.to_string());
            }
        } else {
            quote! {
                let mut __ctx = ::act_sdk::ActContext::__new((), __default_lang.to_string());
            }
        }
    } else {
        quote! {}
    };
```
